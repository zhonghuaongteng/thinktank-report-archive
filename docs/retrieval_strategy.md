# 国际科技智库检索与识别策略

## 目标边界

检索目标以“支撑科技创新发展”为主轴，优先保留能够解释研发体系、创新能力、产业化路径、技术基础设施、科技人才、战略性产业、技术扩散、产业链供应链、创新金融、标准计量测试体系、国防创新和涉华科技竞争的报告、论文、政策简报和高质量分析。

AI治理仍保留专题价值，但默认检索画像不再让纯治理材料挤占主要写入与简报展开位置。需要集中观察AI治理时，使用 `ai_governance_watch` 专题画像。

## 已暴露问题

- 来源结构偏差：GovAI、OECD.AI、Stanford HAI 等源天然推高 AI治理占比，导致产业、科研体系、人才、基础设施和供应链材料显得不足。
- P1识别偏窄：早期 P1 容易被 AI治理、科技治理、涉华政策命中，部分创新支撑型材料因只命中单一主题或非报告页而落到 P2。
- 纯治理噪音：只讨论模型问责、治理流程、一般监管立场的材料，会占据 P0/P1重点区，但对科技创新支撑体系的解释力有限。
- 词语误报：单个 `AI`、`technology`、`regulation`、`data policy`、`national security`、`defense technology` 等词曾造成弱相关候选被抬高。
- 日期误判：sitemap `lastmod` 可能只是页面更新日期，不等于报告发布日期；旧报告被站点重新更新后不能直接进入近三年回填。
- 页面类型噪音：活动页、播客、图书发布、获奖公告、证词、媒体转载、人物页、索引页会污染候选池。
- 来源访问限制：Gartner 等商业研究源只能保存元数据、链接、短摘要和自有研判；403、超时、PDF不可用需要记录，不应阻断简报。
- 批量扫描不稳：全批次只读评估容易超时；后续应按机构小批量评估，避免一次性全源扫描。
- 中文补写副作用：中文摘要中出现“上海”等研究者研判表述时，不应触发重新评分，避免把中文研判误当作原文涉沪信号。
- 摘要过短：每篇报告只写一句中文摘要，无法支撑后续调用。后续中文摘要必须至少回答核心观点、建议和中国/上海参考三项。
- 周报呈现偏短：周频抓取不应沿用日报篇幅，重点条目、专题列表和索引上限需要单独放宽。
- 漫画导读缺位：漫画不应停留在策略表述中，周报需要在正文前预留1-3页视觉导读位置，样式确认前先以样张和提示词固化口径。第一版四格样张和 v2 议题诊断图暴露出议题关键点不够显性的问题，后续样张优先采用“政策冲突图解”结构。

## 识别规则

- 默认画像：`broad_innovation_support`。
- 专题画像：`ai_governance_watch` 仅用于治理专题复核。
- 近三年回填：回填窗口以运行日向前3年为硬边界；不再无止境抓取历史材料。
- 周频增量：默认周日下午14:00运行，周频增量回看14天，依靠 URL 状态库去重。
- 主题保留：科技创新、半导体、先进制造、数字经济、科技人才、国防AI直接视为创新支撑主题。
- 治理例外：科技治理只有在同时出现创新支撑型信号时进入默认画像，例如技术政策、促进创新的监管、监管沙盒、创新采购、标准基础设施、计量、合格评定、测试验证、公共算力、数据互操作、技术扩散、技术采用。
- 纯治理降权：只命中 AI治理或缺少创新支撑信号的科技治理候选，不进入默认宽口径画像；必要时保留到专题观察。
- 涉华/涉沪：作为背景信号处理，只有与实质科技主题共同出现时进入 P0/P1；单独涉华外交、活动或一般政策页面最高保留 P2。
- 报告优先：报告、论文、政策简报和可访问 PDF 优先；一般新闻或博客只有在命中明确创新支撑主题时才进入。
- 来源配比：有限写入批次至少保留一半名额给创新支撑材料；同一优先级中创新支撑材料排在纯治理材料之前。

## 排除规则

- 排除活动、播客、视频、人物、项目索引、广义主题页和订阅页。
- 非报告类图书发布、获奖公告、证词、媒体转载和专家被采访页最高 P2。
- 只有单个弱词命中的材料保持 P3：一般 `AI`、一般 `technology`、一般 `regulation`、一般 `data policy`、一般 `national security`。
- ORF America 等混合内容源在无 PDF、非报告类型时最高 P2，避免把普通评论抬为重点。
- Gartner 标记为 `commercial_research`，默认只保留元数据、链接、短摘要和自有研判。

## 运行策略

- 每周自动运行：北京时间周日14:00。
- 自动化顺序：确认当前工作分支，`git pull --ff-only`，运行单元测试，执行来源健康审计，运行 `scripts\run_weekly.ps1`，核验周报、归档、知识库索引和状态库，最后提交并推送变更。
- 覆盖审计：恢复历史回填前先读取 `docs/backfill_coverage_audit.md`，优先处理零覆盖、低覆盖和入口质量问题。
- 评估方式：新增机构或规则改动前先执行 `evaluate --institution <slug> --limit <n> --backfill --lookback-years 3 --search-profile broad_innovation_support`。
- 超时处理：全批次评估只作审计，不作为常规入口；常规优化按机构小批量运行。
- 去重依据：URL 状态库为主，同一 URL 重复抓取不生成新文件；元数据变化更新同一记录。
- 知识库写入：所有新增条目同步到 `C:\Users\WINDOWS\OneDrive\知识库\系统\研究知识库\06_数据资产\研报_国际智库抓取索引.csv`。
- 产物位置：周报写入 `briefs/weekly/<年份>/`，历史日简报保留在 `briefs/daily/<年份>/`。

## 摘要与周报深度

- 单篇 Markdown 的 `中文摘要与研判` 固定为三段：`核心观点`、`建议`、`中国/上海参考`。
- `核心观点` 应说明作者的中心判断、问题定义和关键证据，不只改写标题。
- `建议` 应抽取报告对政府、企业、科研机构、国际合作或治理工具的行动主张；若原文没有明确建议，应标明“未检出明确政策建议”并保留后续复核入口。
- `中国/上海参考` 应区分直接涉华/涉沪判断与一般比较参照；没有直接信号时，不强行生成涉沪结论。
- 周报 P0/P1 重点条目直接展开三段摘要。周报默认可展示18条重点、12条科技创新支撑、18条广义创新支撑、12条涉华/涉沪和160条索引。
- 周报正文前保留 `漫画导读`，图片确认后插入1-3页；当前候选提示词保存在 `comic/weekly-tech-watch-v3/`。`comic/weekly-tech-watch-v2/` 和旧版四格样张仅作对照，不作为默认生成风格。
- 漫画导读默认采用“报告信号 -> 真正矛盾 -> 上海追问”的三层结构，每页最多表达4个政策冲突，不再用抽象主题清单替代报告关键点。

## 质量闸门

- 单元测试必须通过。
- 规则变更必须有测试覆盖：至少包括保留规则、排除规则和排序/配额规则。
- 抓取前不得降低全局 P1 阈值来扩大结果；应优先通过画像、主题词、页面类型和来源配比控制质量。
- 详情页日期优先于 sitemap `lastmod`；无可靠发布日期的候选不得进入近三年回填。
- 中文标题和中文摘要补写后不重新评分；评分只基于原始标题、英文摘要、关键词、主题和可信元数据。
- 失败不阻断简报：403、超时、PDF不可访问、外链限制和付费墙写入状态库或审计表。
- 摘要质量核验：P0/P1归档条目不得只保留一句短摘要；缺少建议或中国/上海参考时，应在周报或后续推进中显式暴露，而不是隐藏为空。
